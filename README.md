# AIM-PM-Root-MUSIC: Adaptive Subarray Selection for Coherent DOA Estimation

> **Extending:** Zhang et al., *"A Fast and Robust MUSIC Algorithm for Estimating Multiple Coherent Signals"*, CCISP 2024. [DOI: 10.1109/CCISP63826.2024.10765572](https://doi.org/10.1109/CCISP63826.2024.10765572)

---

## Overview

This repository contains:

1. **Full Python reproduction** of all five algorithms benchmarked in the paper (MUSIC, PM-MUSIC, Root-MUSIC, PM-Root-MUSIC, IM-PM-Root-MUSIC).
2. **AIM-PM-Root-MUSIC** — our original contribution: an *adaptive* version of the paper's IM-PM-Root-MUSIC that automatically selects the optimal number of spatial smoothing subarrays based on the estimated operating SNR.

### Why adaptive L?

The paper (Fig. 5) shows that subarray count `L` has a performance sweet spot:

| L too small | L at optimum | L too large |
|---|---|---|
| Poor coherent decorrelation | Lowest RMSE | Subarray too small, aperture loss |

The paper hardcodes `L=9` for their `M=20` setup. But the optimal `L` depends on SNR:
- **Low SNR** → more subarrays → better spatial averaging → lower RMSE
- **High SNR** → fewer subarrays → larger subarray → better angular resolution

**AIM-PM-Root-MUSIC** estimates the operating SNR from the eigenvalue spread of the received covariance matrix, then selects `L*` using a polynomial root proximity criterion (no ground truth needed).

## Using the Algorithms

```python
import numpy as np
from utils.array_signal import generate_received_signal
from algorithms.adaptive_im_pm_root_music import adaptive_im_pm_root_music
from algorithms.im_pm_root_music import im_pm_root_music

M = 20           # array elements
D = 3            # number of sources
N = 100          # snapshots

X = generate_received_signal(
    thetas_deg=[-30.0, -50.0, -10.0],
    snrs_db=[0.0, -5.0, -10.0],
    M=M, N_snapshots=N, coherent=True
)

# Paper's algorithm (fixed L=9)
doas_paper = im_pm_root_music(X, D, M, L=9)

# Our contribution (adaptive L*)
doas_adaptive, L_star, snr_hat, scores = adaptive_im_pm_root_music(
    X, D, M, return_diagnostics=True
)

print(f"Estimated SNR: {snr_hat:.1f} dB")
print(f"Selected L*: {L_star}")
print(f"DOAs: {sorted(doas_adaptive)}")
```

## AIM-PM-Root-MUSIC: Technical Details

### Adaptive Selection Pipeline

```
Received X (M×N)
      │
      ▼
┌─────────────────┐
│  Estimate SNR   │  eigenvalue spread: σ_D / mean(σ_{D+1..M})
│  from X         │
└────────┬────────┘
         │  SNR_hat
         ▼
┌─────────────────┐
│  SNR → search   │  maps SNR range to candidate L window [L_min, L_max]
│  window policy  │
└────────┬────────┘
         │  candidates L ∈ [L_min, L_max]
         ▼
┌─────────────────┐
│  For each L:    │  unit circle proximity = mean |z_k| - 1|  for D closest roots
│  score = root   │  (no ground truth needed — purely data-driven)
│  proximity      │
└────────┬────────┘
         │  L* = argmin(score)
         ▼
┌─────────────────┐
│  IM-PM-Root-    │  spatial smoothing → propagation operator → polynomial roots
│  MUSIC with L*  │
└─────────────────┘
```

### Complexity

| Algorithm | Covariance | Noise subspace | Root finding |
|---|---|---|---|
| MUSIC | O(NM²) | O(M³) eigendecomp | O(n·M·D) search |
| IM-PM-Root-MUSIC | O(NM²) | O((M-L)·D²) ortho | O((2(M-L)-1)³) |
| **AIM-PM-Root-MUSIC** | O(NM²) | O((M-L*)·D²) ortho | O((2(M-L*)-1)³) · K |

Where K = size of search window (typically 3–5). The overhead is small because the window is narrow.

## Computational Complexity Reference

From Table I of the paper:

| Algorithm | Complexity |
|---|---|
| MUSIC | O(NM²) + O(M³) + O(nMD) |
| Root-MUSIC | O(NM²) + O(M³) + O((2M-1)³) |
| PM-Root-MUSIC | O(NM²) + O(M(M-D)D) + O((2M-1)³) |
| IM-PM-Root-MUSIC | O(NM²) + O((M-L)(M-L-D)D) + O((2(M-L)-1)³) |
| **AIM-PM-Root-MUSIC** | Above × K candidates (K ≤ 5) |
